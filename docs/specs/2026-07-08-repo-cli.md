# Spec: repo CLI

## Branches

- [x] Installer removal — superseded by the vercel `skills` CLI (user: "I think its the npx skills (vercel skills) script that does the installing so maybe I don't need one after all")
- [x] CLI scope — what "something useful" means
- [x] Metadata source — where human-facing summaries live

## Decisions

1. **Installer removed** — `scripts/install.mjs` and `package.json` deleted; quickstart points back at `npx skills add adriancodes/skills`; the layer keeps its one-line `cp` install. _Why:_ full overlap with the maintained vercel CLI; the only unique capability (layer routing) doesn't justify 200 lines. (2026-07-08)
2. **CLI scope** — `scripts/skills.mjs` with `table` (markdown table from frontmatter), `readme` (regenerate the README Skills section between markers; `--check` mode diffs without writing), `check` (agentskills.io spec rules + toolbox rules, exit 1 on failure — delivers the deferred CI-validation improvement). _Why:_ user: "build a cli that generates a table of skills and their description or something useful for this repo." (2026-07-08)
3. **Metadata source** — Each SKILL.md carries spec-legal `metadata: category / summary`; the README one-liners are generated from them, never hand-edited. The work-discipline layer participates via its own frontmatter. _Why:_ one source of truth; README drift becomes impossible instead of unlikely.

## Assumptions

- ASSUMED: CLI design details (subcommand names, marker syntax, validation list) chosen under the "or something useful" latitude; corrections are one message.

## Deferred

- Wiring `check` + `readme --check` into a GitHub Action (needs the repo published first).

## Verification

`verify-work` ran against the CLI: 3 rounds, 17 executed attacks (planted fixtures: name mismatch, oversize description/summary, broken and orphaned reference files, frontmatter absent or truncated, stale README, missing markers, unknown subcommand, double-hyphen names, idempotent re-runs). One finding — raw stack trace on a SKILL.md with no frontmatter — patched (check reports it as a finding; table/readme fail with one clean line); rounds 2–3 dry. `check` also caught one real repo issue on its first run: the layer's display name tripped the skill-name rule, resolved by scoping spec name rules to skills only.
